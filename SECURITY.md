# Security Policy

RepoSeal is a security boundary, so bypasses, identity-confusion errors, unsafe canonicalization, lockfile substitution, sandbox escapes, and commands executed after a block are security vulnerabilities.

Report vulnerabilities privately through GitHub Security Advisories for `carterlasalle/RepoSeal`. Include the affected version, platform, exact command, expected and observed decision, and a minimal non-destructive reproduction. Do not include live credentials or publish weaponized HalluSquatting identifiers before coordinated disclosure.

Version 1.x receives security fixes. Preview builds and modified binaries are unsupported. We target initial acknowledgement within 72 hours and coordinate disclosure based on exploitability and downstream impact.
