# ADR-0002: Evidence before score

- Status: Accepted
- Date: 2026-07-12

Canonical identity is a set of attributable claims from registries, repositories, project manifests, domains, lineage, releases, and provenance. Contradictory authoritative claims produce ambiguity. Risk scoring orders review work but cannot average away canonical mismatch or another invariant block. Popularity and README similarity are supporting signals only.
