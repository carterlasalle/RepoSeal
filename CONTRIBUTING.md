# Contributing

Open an issue before changing stable identity, report, policy, lockfile, or benchmark semantics. Security features require a failing safe fixture first, positive/negative/boundary tests, zero-execution assertion for new blocks, documentation, traceability, and a threat-model update when the boundary changes.

Do not submit active malware, reverse shells, credential theft, persistence, destructive commands, or live attacker infrastructure. Minimize reproductions to inert strings and fake providers. Report exploitable bypasses privately per `SECURITY.md`.

Run every command in `docs/RELEASE_V1.md`. Keep commits scoped as contract/test, implementation, then integration/docs. Apache-2.0 contributions certify you have the right to submit the work.
